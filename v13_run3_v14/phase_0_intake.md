## SMELLING SALTS — THE REFRAME

Tim asked for "V14 — the next version of the Learning Loop." But that framing is dangerously narrow. Here's why:

**What Tim said**: V14 = Learning Loop + Constitutional Memory + HB1000 Operating Manual + Bingo City Trojan Horse, all deployed in one execution.

**What that actually is**: V14 is NOT a "next version of the Learning Loop." The Learning Loop is ONE COMPONENT of V14. V14 is a **DEPLOYMENT PROTOCOL** — an orchestration layer that coordinates four independent systems into a single, repeatable deployment.

The difference matters enormously:

| If V14 = "Learning Loop V14" | If V14 = "Deployment Protocol that uses V13" |
|:----------------------------|:---------------------------------------------|
| V13 gets replaced — all that work is lost | V13 stays intact — proven, certified, stable |
| Every component is tangled together | Each component is independent, testable, replaceable |
| One bug breaks everything | One bug breaks one component — others still work |
| Can't use Learning Loop without Bingo City | Can use any component independently |
| Versioning is confusing (is it Learning Loop 14 or Deployment Protocol 1?) | Clean versioning: Learning Loop stays at V13, Deployment Protocol starts at V1 |

**The Smelling Salt**: V14 should NOT be called "Learning Loop V14." It should be called something like **"The HB1000 Deployment Protocol"** or **"The Initiative Deployment System"** — and it CALLS V13 as one of its components. V13 doesn't get replaced. It gets promoted to a component inside a larger system.

This is the same pattern as the Constitutional Memory System: it didn't replace the Learning Loop — it became a foundation layer that the Learning Loop operates on top of. V14 is the ORCHESTRATION layer that sits on top of everything.

## MOVE 37 ALPHA — THE UNCONVENTIONAL INSIGHT

**What if V14 isn't a protocol at all — it's a FRANCHISE KIT?**

Think about what Tim described: a repeatable system that you "drop into" any initiative, it configures itself based on a Local North Star, deploys governance + evaluation + a beautiful interface, and connects to a source of truth. That's not a protocol. That's a franchise.

McDonald's doesn't give each restaurant a "deployment protocol." They give them a franchise kit: the brand standards (constitutional memory), the operations manual (HB1000), the menu and store design (Bingo City Trojan Horse), the supply chain and accounting system (source of truth database), and the quality inspection process (Learning Loop). Each franchise is configured for its local market (Local North Star) but operates under the same system.

**V14 = The SIC Franchise Kit.**

Each "franchise" (initiative) gets:
- The same governance DNA (constitutional memory)
- The same quality engine (Learning Loop V13)
- The same operations manual (HB1000)
- A locally-configured storefront (Bingo City, aimed at the Local North Star)
- Its own books (source of truth database)

The franchise kit is what makes it repeatable. The Local North Star is what makes each franchise unique. The Prime North Star (Ruby Red / Tim's mission) is the brand — it's what makes them all SIC.
